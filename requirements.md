# 要件定義

## 概要
本アプリは、Slay the Spireのプレイデータを記録・閲覧するためのツールです。React + TypeScript + Viteを使用して開発されています。バックエンドなしで完全ローカルで動作するWebアプリケーションです。アップロードされたデータはローカルで分析します。

## 機能要件

### ページ
1. **runファイルアップロードページ**
   - ユーザーがプレイデータをアップロードできるページ。
   - アップロードされたデータをサーバーに送信し、保存する。

2. **データ閲覧ページ**
   - 保存されたプレイデータを一覧表示するページ。
   - データの詳細を確認できる機能。

## 非機能要件
- **パフォーマンス**: 高速なデータ処理とページ遷移。
- **拡張性**: 新しい機能を容易に追加できる設計。
- **保守性**: コードの可読性とテストの充実。
- **セキュリティ**: データはローカルでのみ処理され、外部サーバーに送信されない。

## 技術スタック
- フロントエンド: React, TypeScript, Vite
- スタイル: SCSS
- Linter: ESLint

## 開発環境
- ローカルでモックサーバーを立ち上げ、データの送受信をテスト可能にする。

## ディレクトリ構成

```
record-the-spire/
├── index.html
├── package.json
├── requirements.md
├── vite.config.ts
├── public/
├── src/
│   ├── assets/
│   │   ├── css/
│   │       ├── foundation/
│   ├── features/
│   │   ├── runs/
│   ├── pages/
```

### ディレクトリ説明
- **index.html**: アプリケーションのエントリーポイントとなるHTMLファイル。
- **package.json**: プロジェクトの依存関係やスクリプトを管理するファイル。
- **requirements.md**: アプリケーションの要件定義を記載したドキュメント。
- **vite.config.ts**: Viteの設定ファイル。
- **public/**: 静的ファイルを格納するディレクトリ。
- **src/**: アプリケーションのソースコードを格納するディレクトリ。
  - **assets/**: アセット（画像やスタイルシート）を格納するディレクトリ。
    - **css/**: スタイルシートを格納するディレクトリ。
      - **foundation/**: 基本的なスタイル（リセットやベーススタイル）を定義。
  - **features/**: アプリケーションの機能ごとにコードを整理するディレクトリ。
  - **pages/**: 各ページコンポーネントを格納するディレクトリ。
